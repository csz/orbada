5.1.1. Uruchomienie programu

Do uruchomienia programu wymagane jest środowisko Java (JRE) w wersji co najmniej 1.6. Można je pobrać tutaj.

Windows
Aby uruchomić program ORBADA w systemie operacyjnym Windows należy posłużyć się skryptem uruchomieniowym orbada.bat znajdującym się w głównym katalogu programu. Jego zawartość powinna wyglądać następująco:
start javaw -Xmx200M -Djava.system.class.loader=pl.mpak.startup.StartupClassLoader -jar startup.jar %1 %2 %3 %4
Parametry startowe:
javaw – program wykonywalny środowiska Java. W zmiennej środowiskowej PATH powinna się znaleźć ścieżka do środowiska Java w wersji co najmniej 1.6. W tym miejscu można podać pełną ścieżkę do programu javaw. Program zostanie uruchomiony w tym środowisku niezależnie od zmiennej środowiskowej PATH.
-Xmx200M - parametr określający maksymalną ilość pamięci która może być zarezerwowana przez program. Więcej parametrów środowiska Java można znaleźć na stronie producenta Java – SUN.
%1 %2 %3 %4 - do programu ORBADA mogą być przekazane cztery parametry.

Linux/Unix
Aby uruchomić program ORBADA w systemie operacyjnym Linux/Unix należy posłużyć się skryptem uruchomieniowym orbada.sh znajdującym się w głównym katalogu programu. Jego zawartość powinna wyglądać następująco:
/usr/java/jdk1.6/jre/bin/java -Xmx200M -Djava.system.class.loader=pl.mpak.startup.StartupClassLoader -jar startup.jar
Parametry startowe:
/usr/java/jdk1.6/jre/bin/java - program uruchomieniowy środowiska Java. Jeśli środowisko Java zostało zainstalowane w innej ścieżce niż podana, ten parametr wymaga zmiany.
-Xmx200M - parametr określający maksymalną ilość pamięci która może być zarezerwowana przez program. Więcej parametrów środowiska Java można znaleźć na stronie producenta Java – SUN.

5.1.2. Parametry wywołania programu

-config:<nazwa pliku>
Parametr pozwala na uruchomienie programu z alternatywną konfiguracją główną. Nazwa powinna wskazywać plik w "katalogu użytkownika/.orbada/config" bez rozszerzenia. Rozszerzenie pliku konfiguracji to .properties.

-log4j:<nazwa pliku xml>
Parametr pozwala wskazać alternatywną konfigurację logger'a programu. Nazwa powinna wskazywać plik w "katalogu użytkownika/.orbada/config".

5.1.3. Konfiguracja do uruchomienia z USB Drive

Program ORBADA oraz środowisko JAVA można skonfigurować tak by cały program, wszystkie jego ustawienia, znajdowały się na urządzeniu USB Drive.
Wymagane minimalne miejsce na dysku przenośnym to 150MB.

W tym celu wymagana jest modyfikacja pliku uruchomieniowego BAT lub SH oraz umieszczenie wszystkich wymaganych plików w jednym, głównym katalogu.

Przykładowy układ katalogów:

  • orbada – główny katalog programu
    • app - katalog do którego należy wgrać wszystkie pliki i biblioteki programu ORBADA
    • jre6 - katalog w którym powinno znaleźć się środowisko JAVA ze strony SUN.
    • usr - katalog w którym będą odkładane wszystkie ustawienia i pliki tymczasowe programu
      • temp - katalog plików tymczasowych
      • home - katalog domowy dla plików ustawień oraz lokalnej bazy danych

Zawartość pliku startowego orbada.bat powinna wyglądać tak:

start ../jre6/bin/javaw -Xmx200M -Duser.home=../usr/home -Djava.io.tmpdir=../usr/temp 
  -Djava.system.class.loader=pl.mpak.startup.StartupClassLoader -jar startup.jar %1 %2 %3 %4

Zawartość pliku startowego orbada.sh powinna wyglądać tak:

../jre6/bin/java -Xmx200M -Duser.home=../usr/home -Djava.io.tmpdir=../usr/temp 
  -Djava.system.class.loader=pl.mpak.startup.StartupClassLoader -jar startup.jar

Ważne jest by wytłuszczone lokalizacje katalogów były zgodne z układem katalogów na USB Drive
